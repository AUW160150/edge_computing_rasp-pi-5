Overview

The Mission

The Brief

Build a cloud-to-edge AI agent platform where developers submit natural language tasks via HTTP and a Claude AI agent interprets, executes shell commands on a physical Raspberry Pi 5, and streams results back in real time.

Constraints

  • Complete in 3 days
  • Real hardware — Pi on a home network
  • Production security posture, no shortcuts
  • Everything as code — Terraform, Docker, K8s
  • No hardcoded secrets, ever

Success Criteria

  • End-to-end flow: browser → GKE → Claude → Pi → stdout
  • Real-time streaming via SSE
  • Secure private tunnel — Pi not internet-exposed
  • Isolated execution — Docker sandbox per command
  • Reproducible — one script provisions a new Pi
  • Observable — structured logs, liveness monitor

Safety & Security

5-Layer Security Model

1

X-API-Key

Developer → GKE. Key stored in GCP Secret Manager.

2

Tailscale ACL

tag:gke → tag:pi:8080 only. Zero open ports on Pi.

3

X-Pi-Token

Agent → Pi API auth. Separate secret, injected at runtime.

4

Docker Sandbox

--network=none · --cpus=0.5 · --user=nobody · --rm

5

Secret Manager

Zero hardcoded credentials. Workload Identity for GKE access.

Why Tailscale over Cloudflare?

Tailscale gives cryptographic node identity (WireGuard keys), tag-based ACLs, and no internet exposure — the Pi only accepts traffic from the GKE pod tagged tag:gke.

Why Docker-in-Docker sandbox?

Claude instructs the Pi to run arbitrary shell commands. Each command runs in an ephemeral container with no network, limited CPU, and as nobody — blast radius fully contained.

Workload Identity Federation

GitHub Actions authenticates to GCP via OIDC token exchange — no service account JSON stored in GitHub secrets. Declared in Terraform, scoped to this repo only.

Developer Onboarding

Getting Started

Prerequisites

  • GCP account with billing enabled
  • Anthropic API key
  • Tailscale account (free tier is fine)
  • Raspberry Pi 5 + SD card + power supply
  • GitHub account (for CI/CD deploys)
HARD
1 — Provision GCP infrastructure
Run terraform apply in terraform/. Creates GKE cluster, Artifact Registry, IAM, Secret Manager secrets. Requires GCP familiarity.
MED
2 — Set 5 secrets in Secret Manager + Tailscale ACLs
Paste API keys into GCP console. Configure Tailscale tag-based ACL grant (tag:gke → tag:pi:8080). Generate tagged auth keys.
EASY
3 — Flash Pi OS + run setup.sh
Use Pi Imager to flash headless OS with SSH + WiFi pre-configured. Then: ./setup.sh <PI_TOKEN> <TS_KEY> — installs Docker, joins Tailscale, starts Pi API in ~5 min.
EASY
4 — Push to main → CI/CD deploys automatically
GitHub Actions builds the GKE image, pushes to Artifact Registry, and rolls out via kubectl set image. No manual kubectl needed.
EASY
5 — Open dashboard, type a task, watch it run
Open dashboard/index.html, enter your API key, type any natural language task. Results stream live within seconds.

Improvements Needed

What production deployment requires

🔴 HTTPS / TLS termination

GCP-managed certificate + Ingress. Currently plain HTTP — X-API-Key in cleartext over the wire.

🔴 Authentication hardening

Replace static API key with OAuth2/JWT. Per-user rate limiting and audit log needed.

🟡 Horizontal scaling

Single worker thread is correct for one Pi. Multi-Pi needs a job queue (Redis/Pub-Sub) and per-Pi worker pools.

🟡 Task persistence

Task state is in-memory — lost on pod restart. Needs Firestore or Cloud Spanner for durability across restarts.

🔵 Memory limits on Pi sandbox

--memory=512m flag is set but unenforced — Pi OS Bookworm uses cgroup v2 but memory accounting is disabled by default. Needs cgroup_enable=memory in kernel boot params.

🔵 Observability stack

Structured logs go to Cloud Logging today. Needs Grafana dashboards, alerting, and distributed tracing (OpenTelemetry).

🔵 Tailscale auth key rotation

Tagged ephemeral auth keys should auto-rotate. Currently set manually as a K8s secret — not yet in Terraform.

🔵 Pi setup idempotency

setup.sh works cleanly for a fresh Pi. Needs re-run safety checks for re-provisioning an existing device.

Roadblocks & Troubleshooting
🖥 Open Live Dashboard

What broke, and how I fixed it

🔥

Terraform deleted the GKE node pool

remove_default_node_pool=true in apply destroyed the existing node pool. Fix: imported all existing GCP resources into Terraform state and re-applied to recreate from scratch.

🌐

/pi/status timed out — SOCKS5 proxy missing

GKE API was making a direct HTTP request to the Pi's Tailscale IP, which isn't routable without the sidecar. Fix: added proxies={"http":"socks5h://localhost:1055"} to the requests call.

🔐

Workload Identity binding referenced wrong repository

IAM member used rodela/rodela-trial-project but actual binding was Asari-AI/rodela-trial-project. Fix: found correct principal from gcloud, updated main.tf and re-imported.

📡

Tailscale ACL format mismatch

Account uses the newer grants format, not legacy acls array — UI rejected the policy silently. Fix: rewrote ACL using grants array with src/dst/ip fields.

📦

GKE deployment stuck on hardcoded image tag

deployment.yaml used :v4 which matched nothing pushed. CI only pushed :git-sha. Fix: deployment.yaml → :latest; CI updated to push both :sha and :latest.
